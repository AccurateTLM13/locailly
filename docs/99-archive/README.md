# Archive

Useful context that is **not** the current source of truth for Locaily.

Archived docs may contain outdated names, old ports, old architecture assumptions, or superseded plans.

**Use running code and current docs as source of truth.**

## When To Read Archive

- Understanding historical naming (Local AI Platform, Local AI Engine)
- Recovering detailed engine-core specs from early planning
- Reviewing conversation captures that shaped Pit Crew / track orchestration
- Model/hardware research notes (not implementation truth)

## When Not To Read Archive First

- Implementing features → [../00-start-here/current-state.md](../00-start-here/current-state.md)
- What to build next → [../07-progress/next-agent-brief.md](../07-progress/next-agent-brief.md)
- Track system → [../02-track-system/README.md](../02-track-system/README.md)
- API contracts → [../01-architecture/api-contract.md](../01-architecture/api-contract.md)

## Subfolders

| Folder | Contents |
|---|---|
| `old-summaries/` | Superseded top-level plans |
| `raw-conversation-captures/` | Exported conversations |

## Staleness Warning

Archived migration plans list gaps (no `/tasks/run`, no audit log, etc.) that **have since been implemented**. Trust code and [../00-start-here/current-state.md](../00-start-here/current-state.md) over archive files.
